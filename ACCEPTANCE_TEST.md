# 验收测试说明

软件包：wps-read-aloud-zhangjingyao
版本：1.0.14
目标环境：银河麒麟 V10 ARM64 + WPS 2023 for Linux

## 安装测试

```bash
sudo dpkg -i wps-read-aloud-zhangjingyao_1.0.14_arm64.deb
```

预期结果：
- 安装过程无错误退出。
- 后台 `wps-tts.service` 启动并保持运行。
- 安装日志写入 `/var/log/wps-read-aloud-install.log`。
- 覆盖升级后需要重启 WPS，以加载新版 Ribbon。

## 选项卡测试

1. 打开 WPS 文字。
2. 确认顶部出现“文档朗读”选项卡。
3. 确认选项卡只有这些入口：开始朗读、停止朗读、连页朗读、当页朗读、朗读语速、状态检查、关于朗读。
4. 确认“连页朗读”和“当页朗读”互斥，默认选中“连页朗读”。
5. 确认“朗读语速”包含 `0.5x`、`0.75x`、`1x`、`1.5x`、`2x` 五个选项，选择后只有当前项带勾选提示。
6. 确认不存在“未知按钮”弹窗，不存在多项语速同时显示选中。

## 朗读范围测试

- 连页朗读：有光标时从当前光标处朗读到文档末尾；没有可识别光标时从文档开头朗读到文档末尾。
- 当页朗读：有光标时从当前光标处朗读到当前页末尾；没有可识别光标时从当前页开头朗读到当前页末尾。
- 朗读过程中，当前朗读语句应在 WPS 文档中同步选中，并尽量保持可见。
- 停止朗读后，不应继续播放后续句子。

## 性能测试

1. 打开包含多句文本的文档。
2. 点击“开始朗读”。

预期结果：
- 开始后状态栏提示正在预生成前三句音频。
- 前三句预生成完成后开始播放第一句。
- 第一、第二句之间不应出现明显长等待。
- 低性能机器上仍可点击“停止朗读”结束任务。

## 中英文数字混读测试

示例文本：

```text
这是 WPS 2026 read aloud test，版本是 v1.0.14。
```

预期结果：
- 中文正常朗读。
- 英文和数字不被跳过。
- 英文和数字按单字符中文读法朗读，例如 `WPS`、`2026`、`v1.0.14` 会被逐字符读出。

## 弹窗测试

- “状态检查”和“关于朗读”弹窗不应出现黄色三角叹号图标。
- 弹窗标题字号更大并加粗。
- 内容行距舒展，主要字段不换行。
- 关于弹窗说明文件链接可点击打开。

## 异常操作测试

- 连续快速点击“开始朗读”不会产生多段音频同时播放。
- 朗读过程中再次点击“开始朗读”，上一轮应被停止或安全接管。
- 朗读过程中关闭 WPS 后，服务端不应残留长期运行的合成或播放进程。
