# 发布说明

软件包：wps-read-aloud-zhangjingyao
版本：1.0.6
架构：arm64
开发者：zhangjingyao
发布时间：20260515

## 概述

本版本面向银河麒麟 V10 ARM64 和 WPS 2023 for Linux，提供一个完全离线的 WPS 文字朗读加载项。

## 功能

- WPS 顶部新增“文档朗读”选项卡。
- 所有功能放在 WPS 顶部 Ribbon 选项卡中，不再使用右侧任务窗格。
- 朗读选区。
- 朗读全文。
- 语速和音量通过下拉选项设置。
- 按完整语句逐句朗读。
- 支持中文、英文以及中英文混合文档朗读。
- 朗读时在 WPS 文档中同步选中当前语句。
- 朗读到下一句时，同步切换到下一句选区。
- WPS 通常会跟随当前选区滚动，从而保持当前朗读内容在页面中可见。
- 本地服务只监听 `127.0.0.1:19860`。
- 内置 Piper ARM64 运行环境和中文离线语音模型。
- 内置 eSpeak NG ARM64 兜底运行环境和数据文件。
- 安装时自动进行服务健康检查。
- 安装时自动进行语音合成自检。
- 朗读播放改为由本地服务调用系统播放器完成，规避 WPS 内置浏览器自动播放限制。
- 升级安装时会强制重启 `wps-tts.service`，避免加载项与旧版后台服务不匹配。
- 未检测到系统播放器时，使用随包自带 eSpeak NG 直接播放作为兜底。
- 修复 systemd 服务以桌面用户调用 `pw-play/paplay` 时无法读取临时 WAV 文件的问题。
- 系统播放器改为顺序尝试 `pw-play`、`paplay`、`aplay`，如果 PipeWire 返回 `no node available`，会自动切换到下一种播放方式。
- 前端增加按钮连点节流和播放状态保护。
- 服务端合成请求带超时控制，并在停止时清理语音引擎进程。
- 低配置机器保护：限制单次最多处理 1000 句，单句最多 1000 字，并在合成等待时提示用户。
- 安装前检测端口冲突和同名 WPS 加载项冲突。
- 安装/卸载日志：`/var/log/wps-read-aloud-install.log`。
- 第三方组件许可证和交付文档：`/usr/share/doc/wps-read-aloud-zhangjingyao/`。

## 安装

```bash
sudo dpkg -i wps-read-aloud-zhangjingyao_1.0.6_arm64.deb
```

如果安装时 WPS 已经打开，请安装完成后重启 WPS。

## 已知限制

- 目标系统需要已经安装 WPS 2023 for Linux。
- 当前句子选中和自动滚动依赖目标 WPS 版本对 `Range.Select()` 和 `ScrollIntoView()` 的支持情况。
- 音频播放由本地服务调用系统播放器完成，避免 WPS 内置浏览器自动播放限制；用户主要通过 WPS 顶部“文档朗读”选项卡操作。
- 中文和中英文混合语句优先使用中文 Piper 模型。
- 纯英文语句优先使用 eSpeak NG 英文音色兜底朗读。
- 低性能或内存较小的机器上，建议优先使用“朗读选区”，避免一次朗读超长全文。
