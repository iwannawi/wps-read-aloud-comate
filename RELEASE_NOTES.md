# 发布说明

软件包：wps-read-aloud-zhangjingyao
版本：1.0.14
架构：arm64
开发者：zhangjingyao
发布时间：20260516

## 本次变更

- 重构 WPS 顶部“文档朗读”选项卡，只保留 7 个入口：开始朗读、停止朗读、连页朗读、当页朗读、朗读语速、状态检查、关于朗读。
- “连页朗读”和“当页朗读”改为互斥模式，默认选中“连页朗读”。
- “开始朗读”会按当前模式工作：连页朗读读到文档末尾；当页朗读读到当前页末尾。
- 有光标时从当前光标处开始；没有可识别光标时，连页朗读从文档开头开始，当页朗读从当前页开头开始。
- 朗读语速改为 5 档：`0.5x`、`0.75x`、`1x`、`1.5x`、`2x`。
- 语速菜单取消不兼容的菜单内 toggle 控件，改为普通菜单按钮加当前项勾选标签，避免“未知按钮”和多项同时勾选。
- 英文和数字预处理改为逐字符中文读法，避免 fanchen-C 模型跳过英文或数字。
- 开始朗读时服务端先预生成前三句音频，完成后再播放第一句，减少第一句和第二句之间的等待。
- 重新设计状态检查和关于弹窗，去掉弹窗内图标，扩大窗体和字号，增加行距，减少文字拥挤。
- 重新生成单色大图标，提升 WPS Ribbon 中的清晰度。
- 说明链接精简为发布说明、验收测试、第三方声明三类，减少关于弹窗中的密集链接。

## 安装

```bash
sudo dpkg -i wps-read-aloud-zhangjingyao_1.0.14_arm64.deb
```

如果 WPS 已经打开，请安装完成后重启 WPS。

## 已知限制

- 当前页范围识别依赖 WPS 2023 for Linux 对页码定位接口的支持。如果目标 WPS 页码接口不可用，当页朗读会尽量回退到当前光标到文档末尾的可读范围。
- 正在播放中的当前句不会即时变速，新语速从下一句开始生效。
- 音量由系统音量、声卡或桌面环境声音设置控制，加载项内不提供音量调节。
