# 验收测试说明

软件名称：WPS 文档朗读助手
软件包：wps-read-aloud-xc
版本：1.0.21
目标环境：ARM64 麒麟操作系统 + WPS Office 2023 for Linux / WPS Office 2019 for Linux

## 安装测试

```bash
sudo dpkg -i wps-read-aloud-xc_1.0.21_arm64.deb
```

预期结果：
- 安装过程无错误退出。
- 后台 `wps-tts.service` 启动并保持运行。
- 安装日志写入 `/var/log/wps-read-aloud-install.log`。
- 从旧包名升级时不出现 dpkg 文件覆盖冲突。
- 覆盖升级后重启 WPS，可以加载新版“文档朗读”选项卡。

## 选项卡测试

1. 打开 WPS 文字。
2. 确认顶部出现“文档朗读”选项卡。
3. 确认选项卡入口为：开始朗读、停止朗读、朗读方式、朗读语速、状态检查、关于朗读。
4. 未朗读时“停止朗读”置灰不可用。
5. 朗读中“开始朗读”“朗读方式”“朗读语速”“状态检查”“关于朗读”置灰不可用，“停止朗读”可用。
6. “朗读方式”下拉框包含“连页朗读”和“当页朗读”，默认连页朗读。
7. “朗读语速”下拉框包含 `0.75x`、`1x`、`1.2x`、`1.5x`，默认选中 `1.2x`，选择后只有当前项显示勾选提示。
8. 点击任意功能不应出现“未知按钮”弹窗。

## 朗读范围测试

- 连页朗读：有光标时从当前光标处读到文档末尾；没有可识别光标时从文档开始读到文档末尾。
- 当页朗读：有光标时从当前光标处读到当前页末尾；没有可识别光标时从当前页开始读到当前页末尾。
- 朗读过程中，当前朗读语句应在 WPS 文档中同步选中，并尽量保持可见。
- 停止朗读后，不应继续播放后续句子。

## 启动与性能测试

1. 打开包含多句文本的文档。
2. 点击“开始朗读”。

预期结果：
- 弹出紧凑提示窗，显示“朗读服务正在启动，请耐心等待...”。
- 提示窗不因固定倒计时关闭，进入实际播放后自动关闭。
- 服务端按句动态预处理，累计约 100 字即可开始朗读。
- 第一、第二句之间不应出现明显长等待。
- 低性能机器上仍可点击“停止朗读”结束任务。

## 中英文数字混读测试

示例文本：

```text
这是 WPS 2026 read aloud test，版本是 v1.0.21。
```

预期结果：
- 中文正常朗读。
- 英文和数字不被跳过。
- 英文和数字按单字符中文读法朗读，例如 `WPS`、`2026`、`v1.0.21` 会被逐字符读出。
- 逗号、顿号、分号、冒号等句内语义标点处应有自然停顿；双引号、单引号、书名号、括号等成对符号不额外增加停顿。
- 默认 `1.2x` 语速下，句内标准停顿按约 `400ms` 设计，句末追加约 `600ms` 静音；切换其他语速时停顿随语速等比例缩放。

## 弹窗测试

- 状态检查和关于朗读弹窗不出现黄色三角叹号图标。
- 关于朗读标题为“WPS 文档朗读助手”。
- 关于朗读内容包含适用操作系统、适用办公软件、开发者、发布时间、版本、软件包和服务地址。
- 关于朗读说明文件可以在同一弹窗内点击打开，并可返回关于页。
- 默认窗口大小下内容布局紧凑，尽量不出现滚动条。

## 异常操作测试

- 连续快速点击“开始朗读”不会产生多段音频同时播放。
- 朗读过程中关闭 WPS 后，服务端不应残留长期运行的合成或播放进程。
- 网络不可用时仍可离线朗读，服务只访问 `127.0.0.1:19860`。
